# 安卓 UI 方向

## 产品定位

- App 名称：`Shiroha Quiz`
- 主工作目录：`E:/codex/exercise/shiroha_quiz`
- 目标：做一个支持通用题库导入、练习和考试的原生安卓刷题 App
- 优先级：
  - 第一优先：刷题体验
  - 第二优先：导入能力
  - 第三优先：题库管理、记录、导出等辅助能力

## 视觉方向

主风格：

- 采用 `Apple Card Glass`

交互和平台结构：

- 采用 `Material 3 Pro`

考试模式补充：

- 局部加入更专注、更克制的深色专注感

也就是说：

- 整体观感要高级、轻盈、安静、卡片化
- 但导航、弹层、按钮触达和操作节奏仍然要符合安卓原生习惯

## 关键词

- 毛玻璃卡片
- 浅色分层
- 柔和阴影
- 蓝灰强调色
- 大字号标题
- 宽松留白
- 低噪音控件
- 有质感但不浮夸

## 颜色方向

基础色：

- 背景：`#F4F6FB`
- 提升层背景：`#FBFCFE`
- 卡片底：`rgba(255,255,255,0.78)`
- 描边：`rgba(148,163,184,0.22)`
- 主文字：`#101828`
- 次文字：`#667085`
- 主强调色：`#4F7CFF`
- 柔和强调底：`#E7EEFF`
- 成功：`#17B26A`
- 警告：`#F79009`
- 危险：`#F04438`

考试模式扩展：

- 深石板背景
- 更高对比计时器
- 更少装饰元素

## 字体方向

推荐：

- 中文 UI 字体：`MiSans` 或 `HarmonyOS Sans SC`
- 兜底：`Noto Sans SC`

规则：

- 页面标题要大
- 题干要最强视觉层级
- 选项文字紧凑但清楚
- 辅助文案尽量轻，不要抢视线

## 动效方向

- 页面切换：淡入 + 微上浮
- 选项选中：边框和底色快速强化
- 结果出现：分段进入
- 避免过度弹跳和花哨效果

## 导航模型

底部导航 4 项：

- 首页
- 练习
- 导入
- 我的

二级入口：

- 题库管理从首页进入
- 错题本和记录从首页或我的进入
- 考试模式从练习页进入

## 核心页面

### 1. 首页

作用：

- 展示当前题库
- 快速开始
- 展示最近状态和最近使用内容

布局：

- 顶部标题区
- 当前题库主卡
- 快捷动作
- 进度指标卡
- 最近题库 / 最近操作

主卡内容：

- 当前题库名
- 题量
- 题型摘要
- 两个核心按钮：
  - `开始刷题`
  - `导入题库`

快捷动作：

- 顺序练习
- 随机练习
- 错题复习
- 开始考试

指标内容：

- 今日练习数
- 错题待复习
- 最近成绩

风格：

- 浮层白卡
- 柔和背景渐变
- 轻微光晕

### 2. 刷题页

作用：

- 做到沉浸、清晰、顺手

顶部区域：

- 当前题号
- 题型
- 进度
- 模式标签

题目卡：

- 大题干
- 分类或标签
- 清晰选项

选项规则：

- 短选项尽量横向舒展
- 中等长度可两行
- 长选项自动回到整行卡片
- 不允许出现狭窄竖排选项块

交互：

- 已选反馈必须很强
- 提交后显示正确/错误状态
- 解析在提交后展示

底部操作：

- 提交答案
- 查看解析
- 下一题

风格：

- 只突出一张主卡
- 少装饰
- 底部操作区更方便单手点击

### 3. 导入页

作用：

- 强大，但不能显得很技术化

入口分三类：

- 单文件导入
- 双文件导入
- 粘贴文本导入

顶部内容：

- 导入方式切换
- 当前支持格式说明

主要流程：

1. 选择来源
2. 选择识别模式
3. 预览识别结果
4. 必要时修正异常题
5. 确认导入

预览区：

- 总题数
- 识别到的题型统计
- 异常题摘要
- 异常题优先显示

编辑交互：

- 点开某道题
- 进入底部弹层编辑
- 修正题干、题型、选项、答案、解析

风格：

- 更像“高级导入控制台”
- 不像后台表格页

## Android 技术建议

推荐方案：

- Kotlin
- Jetpack Compose
- Room 本地数据库
- SAF 文件选择器
- 本地解析层
- PDF 首期只支持文字层提取

原因：

- 最适合高级质感界面
- 最适合做强刷题体验
- 长期维护比 WebView 壳更好

## 首版范围

必须有：

- 默认 C1 示例题库
- 导入 `txt / json / csv / docx / doc / 文字层 pdf`
- 双文件导入
- 题库列表
- 练习模式
- 考试模式
- 错题本
- 本地记录
