# 安卓页面线框说明

## 范围

这份文档定义 `Shiroha Quiz` 安卓首批最关键的 3 个页面：

- 首页
- 刷题页
- 导入页

目标是先把：

- 信息层级
- 操作顺序
- 视觉主次

全部锁定，再进入更深的实现。

设计基线：

- 主视觉：`Apple Card Glass`
- 平台结构：`Material 3`
- 形式：安卓原生

---

## 1. 首页

### 页面目标

首页必须在 3 秒内回答用户这 3 个问题：

1. 我当前正在用哪个题库？
2. 我现在最该点哪个入口？
3. 我最近练得怎么样？

### 页面结构

从上到下：

1. 顶部安全区标题
2. 品牌/标题区
3. 当前题库主卡
4. 快捷入口区
5. 进度指标卡
6. 最近题库/最近使用

### 线框

```text
+--------------------------------------------------+
```

### 视觉重点

- 当前题库主卡必须是全页视觉中心
- 主按钮必须是 `开始刷题`
- `导入题库` 可见，但明显次一级
- 指标要像“鼓励性反馈”，而不是压迫性的仪表盘

### 交互重点

- 点击当前题库主卡 -> 打开题库切换或题库详情
- 点击最近题库 -> 直接切换当前题库
- 长按快捷入口以后可以扩展更多选项，但首版先不做复杂

## 2. 刷题页

### 页面目标

刷题页要做到：

- 题干容易读
- 选项容易扫
- 已选反馈明显
- 下一步永远明确

### 页面结构

从上到下：

1. 紧凑信息条
2. 题目主卡
3. 选项区
4. 操作区
5. 可展开解析区

### 线框

```text
+--------------------------------------------------+
| [第12题] [单选题] [顺序练习]             12 / 50  |
+--------------------------------------------------+

+--------------------------------------------------+
| 雨天临近弯道行驶时，最稳妥的做法是？              |
|                                                  |
| +----------------------------------------------+ |
| | A | 保持原速，快速通过弯道                   | |
| +----------------------------------------------+ |
|                                                  |
| +==============================================+ |
| | B | 提前减速，平顺转向                  [已选] | |
| +==============================================+ |
|                                                  |
| +----------------------------------------------+ |
| | C | 紧急制动后再打方向                       | |
| +----------------------------------------------+ |
|                                                  |
| [提交答案]    [查看解析]                        |
+--------------------------------------------------+

+--------------------------------------------------+
| 解析预览                                           |
| 正确答案是 B。核心不是“更快通过”，而是先稳定附着、 |
| 再控制方向，所以应该提前减速，避免急刹和急打方向。 |
+--------------------------------------------------+
```

### 选项布局规则

- 短选项尽量一行展示
- 中等长度最多两行
- 较长选项自动转为整行卡片
- 绝不允许出现窄列竖排文字选项

### 已选中状态规则

已选状态必须同时具备：

- 更强的边框
- 更明确的底色
- 高对比字母圆点
- 稍重的文字字重
- 可选 `已选` 标签

### 按钮优先级

提交前：

- 主按钮：`提交答案`
- 副按钮：`查看解析`

提交后：

- 主按钮：`下一题`
- 副按钮：`再看解析`

### 解析规则

- 只在提交后或主动查看后显示
- 必须明确给出正确答案
- 先短解释，再展开详细解释
- 不要一上来塞大段长文

## 3. 导入页

### 页面目标

导入页要让用户觉得：

- 我知道要传什么
- 这 App 支持很多格式
- 就算识别错了，我也能修

### 页面结构

从上到下：

1. 页面标题
2. 导入方式切换
3. 上传面板
4. 支持格式标签
5. 识别结果摘要
6. 异常题优先列表

### 线框

```text
+--------------------------------------------------+
| Shiroha Quiz                                     |
| 导入题库                                         |
| 首发支持 txt / json / csv / docx / doc / 文字层PDF |
+--------------------------------------------------+

+--------------------------------------------------+
| [单文件导入] [双文件导入] [粘贴文本]              |
+--------------------------------------------------+

+--------------------------------------------------+
|                 [上传图标]                        |
|                选择题库文件                       |
|      拖拽/点击选择文件，优先支持结构化文本         |
+--------------------------------------------------+

  [TXT] [JSON] [CSV] [DOCX] [DOC] [PDF]

+--------------------------------------------------+
| 识别预览                                           |
| 本次识别到 420 题，其中 17 题需要人工确认          |
+--------------------------------------------------+

+--------------------------------------------------+
| [第12题] [单选题]                                 |
| 机动车驾驶证实习期是多长时间？                    |
| 答案 D                                            |
+--------------------------------------------------+

+--------------------------------------------------+
| [第14题] [待确认]                                 |
| 题干与答案文件序号未完全匹配                      |
| 需要人工确认                                      |
+--------------------------------------------------+

               [确认导入题库]
```

### 导入方式

#### 单文件导入

适用于：

- txt
- json
- csv
- docx
- doc
- 文字层 pdf

页面重点：

- 一个干净的上传区
- 高级解析选项默认收起

#### 双文件导入

适用于：

- 题目文件 + 答案文件

页面重点：

- 两个上传卡片
- 小屏竖排，大屏可并排
- 下方放对应策略选择

#### 粘贴文本导入

适用于：

- 直接粘贴题库文本

页面重点：

- 大输入框
- 识别模式放在附近

### 预览规则

- 先给简洁统计
- 异常题优先显示
- 点开某题进入修正弹层
- 修正弹层首版建议用底部弹层，不要跳太深页面

### 人工修正弹层字段

- 题型
- 题干
- 选项
- 答案
- 解析
- 分类

操作：

- 保存
- 删除
- 忽略

### 页面原则

导入页不能像后台表格页。

更应该像：

- 高级导入控制台
- 主流程简单
- 异常修正清楚

## 共用组件

这三页必须共享同一套小组件体系：

- 毛玻璃主卡
- 指标卡
- 胶囊标签
- 已选选项卡
- 上传面板
- 提示卡
- 底部导航

这样整体观感才统一。

---

## 三页优先级总结

### 最重要页面

`刷题页`

原因：

- 它是整个产品体验的中心

### 最重要转化点

`导入 -> 预览 -> 确认`

原因：

- 导入体验决定用户是否信任这款通用刷题器

### 最重要信任点

`首页当前题库主卡`

原因：

- 用户必须随时明确自己现在练的是哪一套题
