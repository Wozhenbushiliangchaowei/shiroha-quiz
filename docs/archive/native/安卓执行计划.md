## 当前阶段（历史记录）

我们现在已经有：

- 产品方向
- 视觉方向
- 首批关键页面线框
- 通用网页版作为功能参考

下一步不适合直接一头扎进完整功能开发，而是要先把“可执行的产品和界面骨架”定稳。

## 推荐下一步

最合适的下一步不是立刻把所有功能做完，而是按下面顺序推进：

1. 锁定安卓 App 的信息架构
2. 定义可复用组件
3. 定义页面状态和关键流程
4. 再进入 Compose 原生界面实现

这样能避免页面和交互反复推倒重来。

## 风险提示

### 风险 1：太照搬网页版

如果把网页版结构 1:1 复制到原生安卓里，最后体验还是会像“套壳工具”。

规则：

- 复用数据结构和解析经验
- 不照搬 Web 页面结构和操作方式

### 风险 2：导入页太像后台

导入能力很强，但很容易做成管理系统味很重的界面。

规则：

- 默认流程简单
- 高级解析选项只在需要时展开

### 风险 3：刷题页太花

如果卡片、毛玻璃、层次做太多，刷题页会失去专注感。

规则：

- 保持一个主卡片为视觉中心
- 选中反馈强
- 顶部和底部操作尽量简洁
